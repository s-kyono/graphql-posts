# UI 思想・コンポーネント戦略

## スタック

| 役割                 | ライブラリ          | 理由                                                      |
|----------------------|---------------------|-----------------------------------------------------------|
| ヘッドレス UI        | **Radix UI**        | アクセシビリティ・キーボード操作・WAI-ARIA が標準装備      |
| スタイリング         | **Tailwind CSS v4** | スタイルの完全コントロール、クラス名で完結                |
| コンポーネント集     | **shadcn/ui**       | Radix + Tailwind のベスト実装。コードを自分でコピー所有   |
| アニメーション       | **Motion**          | 必要なときだけ導入。CSS `@starting-style` で代替できる場合は不要 |
| 大量リスト           | **TanStack Virtual** | 仮想スクロールが必要な場面で                             |

shadcn/ui はパッケージとしてインストールするのではなく、コードをコピーして自分のプロジェクトに取り込む。コードの所有権は自分にある。

---

## UI 設計原則（Ant Design の思想を参考に）

### 1. 自然（Natural）
ユーザーの直感に従う動作を優先する。慣れ親しんだインタラクションパターンを壊さない。

### 2. 確実（Certain）
操作の結果が予測できること。ボタンを押したら何が起きるかが明確。フィードバック（ローディング・成功・エラー）を必ず返す。

### 3. 意味のある（Meaningful）
すべての UI 要素に存在意義を持たせる。「とりあえず置いた」要素は作らない。情報の優先度をビジュアル階層で表現する。

### 4. 成長する（Growing）
コンポーネントは再利用・拡張を前提に設計する。Props で柔軟性を担保しつつ、デフォルト値でゼロコストで使えるようにする。

## コンポーネント設計

### 構成単位

```
pages/        ← ルーティング単位、レイアウトの組み合わせ
features/     ← ドメインロジックを含むコンポーネント
components/   ← ロジックを持たない汎用 UI（shadcn ベース）
```

### Props 設計の方針

- デフォルト値を充実させ「ゼロ設定で動く」を目指す
- スタイルの上書きは `className` で受け取る（`cn()` でマージ）
- 複雑なバリアントは `cva`（class-variance-authority）で管理

```tsx
const Button = ({ variant = "default", size = "md", className, ...props }) => (
  <button className={cn(buttonVariants({ variant, size }), className)} {...props} />
);
```

## アクセシビリティ

Radix UI を使うことで以下が自動で担保される。

- キーボード操作（Tab / Enter / Escape / 矢印キー）
- スクリーンリーダー対応（aria 属性）
- フォーカス管理（モーダル開閉時のフォーカストラップ）

追加で必要なこと：

- `alt` テキストは必須（空画像は `alt=""`）
- カラーコントラスト比 **4.5:1** 以上（WCAG AA）
- タッチターゲット最小 **48px**（→ [responsive-rules.md](./responsive-rules.md)）

## アニメーション方針

| ケース                         | 手法                        |
|--------------------------------|-----------------------------|
| フェードイン・スライドなど単純なもの | CSS `@starting-style` + `transition` |
| ドラッグ・複雑なシーケンス       | **Motion**                  |
| リスト並び替えアニメーション     | **Motion** の `layout` prop |

アニメーションは **目的を持たせる**。「動いてるだけ」の装飾アニメは入れない。
ユーザーの操作に対するフィードバックか、状態変化の文脈を伝えるためにだけ使う。

---

## やってはいけないこと

- Ant Design をそのまま使う（パフォーマンス・スタイル上書きのコスト）
- スタイルのために Radix をフォーク・改造する（shadcn のコードを直接編集して対応）
- アニメーションライブラリを何個も並列導入する
- `z-index` をマジックナンバーで管理する（Tailwind の `z-*` クラスで統一）
